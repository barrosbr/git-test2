0 - Iniciar o track do git em um diretório e todos os sub-diretorios
	git init

1 - clonar um repositorio:
	git clone git@github.com:FilipeOliiveira/Portfolio_Next.git
	
	1.1 - git push origin main -> envia para o github a branch main do repositorio local origin 

2 - git log -> ver os commits

	2.1 Mostra detalhes dos commits
		git log -p
		
	2.2 Mostra um linha para cada commit
		git log --oneline

33 - Baixar commits para o repositorio local. 
		git pull [repositório remoto] [branch main repositório local]
		git pull origin main

4 - listar as entradas remotas 
	git remote -v 

5 - adicionar um repositorio
	- criar uma nova pasta e excetuar:
		git remote add gittest git@github.com:barrosbr/git-test.git
		
		, foi criado e linkado o repositorio local gittest com o repositorio remoto git@github.com:barrosbr/git-test.git
	
	5.1 - Remover um repositorio
		git remote remove origin
		
	5.2 - Alterar a URL de um repositório remoto
		git remote set-url origin https://github.com/seu-usuario/seu-repositorio
	
	5.3 - Alterar o apelido de um repositório remoto
		git remote rename origin novo-origin

8 - Apagando um commit (escolher o id do commit anterior, será apagado os commit apos o id inserido no comando:
	git reset --hard 7cee45367f063560d3532836094ba227e82bc936 -> vai apagar os commits posteriores ao 7cee45367f063560d3532836094ba227e82bc936

13 - Salvar alterçẽos sem fazer commit (guarda uma alteração para que possa ser usada depois):
	git stash
	
14 - Voltar para o ponto da alteração que não foi commitada (é uma pilha pega o stash mas recente):
	git stash pop
	
15 - Listar stash
	git stash list
	
16 - Apagar stash
	git stash clear
	
17 - stash com nome
	git stash push -m "Movendo chamada de funcao"
	
18 - seleciona o numero do stach:

20 - Faz um ctrl z nos arquivos da working tree (faz a mesma coisa: git checkout -- .):

21 - Retirar um arquivo (app.js) da staging area
	git restore --staged app.js

	git restore --source=9e938b8f7747cfa4c500adbd06009c2435714eb2 index.html
